并行消息

显然是可以嵌套的

    actor client1
    actor client2

        server->>client1: response
    and 响应用户2
        server->>client2: response
    end
    client1-->>server: request next

结果可预见的情况，列举所有情况

以一个服务跟数据库建立连接为例，结果无非：

  • 连接成功
  • 连接超时
  • 连接被拒绝

critical … option … option … end

sequenceDiagram
    critical 良好的连接环境
        service-->DB: 连接成功
    option 网络超时
        service-->service: 登录失败
    option 密码错误
        service->>DB: 登录验证
        DB->>service: 验证失败
